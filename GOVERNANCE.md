# Project Governance

This document describes the governance model for the Assistants project.

## Overview

Assistants is an open source project maintained by Hasna. We welcome contributions from the community and aim to make the contribution process as transparent as possible.

## Roles and Responsibilities

### Maintainers

Maintainers are responsible for:
- Reviewing and merging pull requests
- Managing releases
- Setting project direction and priorities
- Enforcing the code of conduct
- Making final decisions on technical matters

Current maintainers:
- Hasna ([@hasna](https://github.com/hasna)) - Project Lead

### Contributors

Contributors are anyone who:
- Submits pull requests
- Reports issues
- Participates in discussions
- Improves documentation
- Helps other users

## Decision Making

### Technical Decisions

Technical decisions are made through:

1. **Discussion** - Proposals are discussed in GitHub issues or pull requests
2. **Consensus** - We aim for consensus among maintainers
3. **Maintainer Decision** - If consensus cannot be reached, maintainers make the final call

### Feature Requests

New features are evaluated based on:
- Alignment with project goals
- Community demand
- Implementation complexity
- Maintenance burden

### Breaking Changes

Breaking changes require:
- Clear documentation of the change
- Migration path for existing users
- Adequate notice period when possible

## Contribution Process

1. Fork the repository
2. Create a feature branch
3. Make changes following our guidelines
4. Submit a pull request
5. Address review feedback
6. Merge upon approval

See [CONTRIBUTING.md](./CONTRIBUTING.md) for detailed guidelines.

## Communication

- **GitHub Issues** - Bug reports, feature requests, discussions
- **Pull Requests** - Code review and technical discussion

## Releases

Releases follow semantic versioning:
- **Major** - Breaking changes
- **Minor** - New features (backward compatible)
- **Patch** - Bug fixes (backward compatible)

## Amendments

This governance document may be amended by maintainers. Significant changes will be communicated to the community.
